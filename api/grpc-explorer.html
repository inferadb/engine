Getting Started with gRPC Explorer

üìã Prerequisites Make sure InferaDB is running locally: 
docker run -p 8080:8080 -p 8081:8081 inferadb/inferadb-engine:latest
The gRPC server runs on port 8081 by default.
1 Install grpcui 
# macOS
brew install grpcui

# Linux/Windows (using go)
go install github.com/fullstorydev/grpcui/cmd/grpcui@latest
2 Start the gRPC UI
grpcui -plaintext localhost:8081

This will open an interactive web UI in your browser at http://localhost:8080 (or a different port if 8080 is taken).

3 Explore the API

The gRPC UI provides an interactive form for each RPC method. You can:

  • Select a method from the dropdown
  • Fill in the request fields
  • Add authentication metadata (if needed)
  • Invoke the method and see the response
üîê Authentication If authentication is enabled, add your JWT token as metadata:
  • Click "Add metadata"
  • Name: authorization
  • Value: Bearer <your-jwt-token>
See the Authentication Guide for creating JWTs.

Health Check

Check if the gRPC server is running and healthy.

Request

{
  // Empty request
}

Response

{
  "status": "healthy",
  "service": "inferadb-engine"
}

Using grpcurl

grpcurl -plaintext localhost:8081 infera.v1.InferaService/Health

Check Permission

Check if a subject has a specific permission on a resource.

Request

{
  "subject": "user:alice",
  "resource": "document:readme",
  "permission": "viewer"
}

Response

{
  "decision": "DECISION_ALLOW"  // or "DECISION_DENY"
}

Using grpcurl

grpcurl -plaintext -d '{"subject":"user:alice","resource":"document:readme","permission":"viewer"}' localhost:8081 infera.v1.InferaService/Check

Expand Relation

  "relation": "viewer"

grpcurl -plaintext -d '{"resource":"document:readme","relation":"viewer"}' localhost:8081 infera.v1.InferaService/Expand

Write Tuples

Write one or more authorization tuples to the store.

Request

{
  "tuples": [
    {
      "object": "document:readme",
      "relation": "viewer",
      "user": "user:alice"
    },
    {
      "object": "document:readme",
      "relation": "editor",
      "user": "user:bob"
    }
  ]
}

Response

{
  "revision": "1234567890",
  "tuples_written": 2
}

Using grpcurl

grpcurl -plaintext -d '{"tuples":[{"object":"document:readme","relation":"viewer","user":"user:alice"}]}' localhost:8081 infera.v1.InferaService/Write

Delete Tuples

Delete one or more authorization tuples from the store.

Request

{
  "tuples": [
    {
      "object": "document:readme",
      "relation": "viewer",
      "user": "user:alice"
    }
  ]
}

Response

{
  "revision": "1234567891",
  "tuples_deleted": 1

Using grpcurl

grpcurl -plaintext -d '{"tuples":[{"object":"document:readme","relation":"viewer","user":"user:alice"}]}' localhost:8081 infera.v1.InferaService/Delete

Alternative Tools

grpcurl

A command-line tool for interacting with gRPC servers (like curl for gRPC).

# Install
brew install grpcurl  # macOS
go install github.com/fullstorydev/grpcurl/cmd/grpcurl@latest

# List services
grpcurl -plaintext localhost:8081 list

# List methods
grpcurl -plaintext localhost:8081 list infera.v1.InferaService

# Describe a method
grpcurl -plaintext localhost:8081 describe infera.v1.InferaService.Check

# Invoke a method
grpcurl -plaintext -d '{"subject":"user:alice","resource":"doc:1","permission":"viewer"}' \
  localhost:8081 infera.v1.InferaService/Check

BloomRPC

A GUI client for testing gRPC services (like Postman for gRPC).

Download from: github.com/bloomrpc/bloomrpc

  • Import the infera.proto file
  • Set the server address to localhost:8081
  • Select a method and fill in the request
  • Click "Play" to invoke the method

Postman

Postman now supports gRPC APIs natively.

  • Create a new gRPC request
  • Enter server URL: localhost:8081
  • Import the proto file or use server reflection
  • Select a method and send requests
